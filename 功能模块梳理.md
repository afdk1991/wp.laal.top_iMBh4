# 多端统一功能模块梳理

## 一、核心功能列表

### 1. 登录注册
- **功能描述**：用户通过手机号、邮箱或第三方账号登录/注册
- **统一字段名**：
  - `phone`：手机号
  - `email`：邮箱
  - `password`：密码
  - `code`：验证码
  - `nickname`：昵称
  - `avatar`：头像
- **交互逻辑**：
  - 登录：输入账号密码或验证码登录
  - 注册：输入手机号/邮箱，获取验证码，设置密码，完成注册
  - 第三方登录：微信、QQ、Google、Facebook等
- **业务规则**：
  - 手机号/邮箱格式验证
  - 密码强度验证
  - 验证码有效时间5分钟
  - 登录失败次数限制

### 2. 用户中心
- **功能描述**：用户个人信息管理、收货地址管理、账户安全设置
- **统一字段名**：
  - `user_id`：用户ID
  - `nickname`：昵称
  - `avatar`：头像
  - `phone`：手机号
  - `email`：邮箱
  - `gender`：性别
  - `birthday`：生日
  - `addresses`：收货地址列表
- **交互逻辑**：
  - 查看个人信息
  - 修改个人信息
  - 管理收货地址
  - 修改密码
  - 绑定/解绑第三方账号
- **业务规则**：
  - 个人信息修改需要验证密码
  - 收货地址最多保存20个
  - 至少设置一个默认地址

### 3. 列表展示
- **功能描述**：商品列表、订单列表、消息列表等
- **统一字段名**：
  - `list`：数据列表
  - `total`：总条数
  - `page`：当前页码
  - `page_size`：每页条数
  - `search`：搜索关键词
  - `filter`：筛选条件
- **交互逻辑**：
  - 下拉刷新
  - 上拉加载更多
  - 搜索
  - 筛选
  - 排序
- **业务规则**：
  - 分页默认每页10条
  - 搜索关键词长度限制
  - 筛选条件组合规则

### 4. 详情页
- **功能描述**：商品详情、订单详情、用户详情等
- **统一字段名**：
  - `id`：详情ID
  - `name`：名称
  - `description`：描述
  - `price`：价格
  - `stock`：库存
  - `images`：图片列表
  - `attributes`：属性
- **交互逻辑**：
  - 查看详情
  - 分享
  - 收藏
  - 购买/下单
- **业务规则**：
  - 详情数据缓存策略
  - 权限验证（部分详情需要登录）

### 5. 支付
- **功能描述**：订单支付、充值等
- **统一字段名**：
  - `order_id`：订单ID
  - `amount`：金额
  - `payment_method`：支付方式
  - `transaction_id`：交易ID
  - `status`：支付状态
- **交互逻辑**：
  - 选择支付方式
  - 确认支付
  - 支付结果回调
- **业务规则**：
  - 支付超时时间30分钟
  - 支付方式：微信支付、支付宝、银行卡等
  - 支付状态更新机制

### 6. 订单
- **功能描述**：订单创建、查询、管理
- **统一字段名**：
  - `order_id`：订单ID
  - `user_id`：用户ID
  - `amount`：订单金额
  - `status`：订单状态
  - `create_time`：创建时间
  - `pay_time`：支付时间
  - `goods`：商品列表
- **交互逻辑**：
  - 创建订单
  - 查看订单列表
  - 查看订单详情
  - 取消订单
  - 确认收货
  - 评价订单
- **业务规则**：
  - 订单状态流转：待支付 → 待发货 → 待收货 → 已完成 → 已取消
  - 订单超时自动取消
  - 订单评价规则

### 7. 数据统计
- **功能描述**：用户数据、订单数据、销售数据等统计
- **统一字段名**：
  - `stat_type`：统计类型
  - `start_date`：开始日期
  - `end_date`：结束日期
  - `data`：统计数据
- **交互逻辑**：
  - 选择统计时间范围
  - 查看统计图表
  - 导出统计数据
- **业务规则**：
  - 统计数据权限控制
  - 数据缓存策略

## 二、统一接口规范

### 1. 接口命名规范
- 采用 RESTful API 风格
- 接口路径：`/api/v1/{module}/{action}`
- 例如：`/api/v1/user/login`、`/api/v1/order/list`

### 2. 请求参数规范
- GET 请求：参数通过 URL 查询字符串传递
- POST/PUT/DELETE 请求：参数通过请求体传递，格式为 JSON
- 公共参数：
  - `token`：用户认证令牌
  - `timestamp`：请求时间戳
  - `sign`：签名

### 3. 响应格式规范
```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 响应数据
  },
  "timestamp": 1620000000000
}
```

### 4. 错误码规范
- 200：成功
- 400：请求参数错误
- 401：未授权
- 403：禁止访问
- 404：资源不存在
- 500：服务器内部错误

## 三、统一业务规则

### 1. 权限验证
- 所有需要登录的接口必须验证 `token`
- 管理员接口需要验证管理员权限
- 敏感操作需要二次验证

### 2. 数据验证
- 所有请求参数必须进行验证
- 验证规则：必填、格式、长度、范围等
- 验证失败返回 400 错误

### 3. 数据安全
- 密码加密存储
- 敏感数据传输加密
- SQL 注入防护
- XSS 攻击防护

### 4. 性能优化
- 接口响应时间 < 300ms
- 数据缓存策略
- 批量操作支持
- 分页查询优化

## 四、多端适配策略

### 1. 响应式设计
- PC 端：响应式布局，1200px 居中
- 移动端：750rpx 标准设计稿
- 平板：双列布局、自适应栅格

### 2. 交互适配
- PC 端：支持键盘快捷键、鼠标 hover
- 移动端：支持触摸操作、手势
- 小程序：支持微信授权、转发

### 3. 性能适配
- PC 端：优先加载完整功能
- 移动端：优先加载核心功能，非核心功能懒加载
- 小程序：控制包体积，分包加载

## 五、技术实现要点

### 1. 后端实现
- 使用 Node.js + Express 构建
- MySQL 数据库存储
- Redis 缓存
- JWT 认证

### 2. 前端实现
- 使用 uni-app 跨端框架
- Vue 3 + Vite 构建
- Pinia 状态管理
- Tailwind CSS 样式

### 3. 多端构建
- PC 网站：H5 网页，直接部署
- 移动端 APP：Capacitor 打包 Android/iOS
- 小程序：微信/支付宝/抖音小程序
