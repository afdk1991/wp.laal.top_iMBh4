检查脚本

#!/bin/bash
# 服务器资源检查脚本

echo "=== 服务器资源检查 ==="

# CPU 检查
echo "CPU 核心数: $(nproc)"

# 内存检查
echo "内存总量: $(free -h | grep Mem | awk '{print $2}')"
echo "内存使用: $(free -h | grep Mem | awk '{print $3}')"

# 磁盘检查
echo "磁盘空间:"
df -h | grep -v tmpfs

# 网络检查
echo "网络连接:"
netstat -i | tail -n +3

# 系统版本
echo "操作系统版本:"
cat /etc/os-release | head -3

# 示例 Deployment 配置检查项

kind: Deployment
metadata:
  name: user-service
  namespace: mixla-prod

  replicas: 3                    # ✅ 检查副本数 ≥ 2
  selector:
    matchLabels:
      app: user-service
  strategy:
    type: RollingUpdate         # ✅ 使用滚动更新
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    spec:
      containers:
        - name: user-service
          resources:
            requests:           # ✅ 检查资源配置

              memory: "512Mi"

              cpu: "2000m"
              memory: "2Gi"

              path: /health

            periodSeconds: 10
          readinessProbe:       # ✅ 检查就绪探针

              path: /ready

四、安全检查

4.1 网络安全

检查项

检查项 检查方法 标准
网络策略 检查 NetworkPolicy 默认拒绝，精确放行
防火墙规则 iptables -L 仅开放必要端口
SSL/TLS 检查证书配置 TLS 1.2+
HTTPS 强制 HTTPS HTTP 自动跳转

网络策略示例

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: user-service-network-policy
  namespace: mixla-prod
spec:
  podSelector:
    matchLabels:
      app: user-service
  policyTypes:
    - Ingress
    - Egress
  ingress:
    - from:
        - namespaceSelector:
            matchLabels:
              name: mixla-prod
        - podSelector:
            matchLabels:
              app: api-gateway
      ports:
        - protocol: TCP
          port: 8080
  egress:
    - to:
        - podSelector:
            matchLabels:
              app: mysql
      ports:
        - protocol: TCP
          port: 3306

检查清单

  • [ ] Kubernetes 网络策略已配置
  • [ ] 默认命名空间策略为拒绝
  • [ ] 服务间通信受网络策略控制
  • [ ] 外部访问仅通过 Ingress
  • [ ] 数据库端口仅内网可访问
