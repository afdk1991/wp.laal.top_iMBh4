# 访问端统一适配指南

## 1. 响应式设计原则

### 1.1 设计理念
- **移动优先**：先设计移动端，再逐步适配更大屏幕
- **断点系统**：使用统一的断点定义，确保所有页面行为一致
- **流体布局**：使用相对单位，避免固定宽度
- **弹性网格**：使用CSS Grid和Flexbox实现灵活布局
- **响应式组件**：确保所有组件在不同设备上表现一致

### 1.2 断点定义

| 断点名称 | 屏幕宽度 | 设备类型 | 网格列数 |
|---------|----------|---------|----------|
| xs | < 480px | 小屏幕手机 | 1 |
| sm | 480px - 640px | 手机 | 2 |
| md | 640px - 768px | 平板 | 3 |
| lg | 768px - 1024px | 小笔记本 | 4 |
| xl | 1024px - 1280px | 笔记本 | 6 |
| 2xl | > 1280px | 桌面 | 8 |

## 2. 统一布局规范

### 2.1 容器使用
```html
<!-- 使用统一的容器类 -->
<div class="container">
  <!-- 内容 -->
</div>
```

### 2.2 网格系统
```html
<!-- 响应式网格 -->
<div class="grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4">
  <!-- 网格项 -->
</div>
```

### 2.3 导航适配
- **桌面端**：顶部导航栏 + 侧边菜单
- **移动端**：顶部导航栏 + 底部导航栏
- **响应式显示**：使用 `desktop-nav` 和 `mobile-nav` 类

## 3. 组件适配

### 3.1 按钮
- **触摸目标**：移动端最小触摸目标 44x44px
- **尺寸适配**：在小屏幕上使用更小的按钮尺寸
- **布局适配**：在小屏幕上垂直堆叠按钮

### 3.2 表单
- **输入框**：在小屏幕上使用全宽输入框
- **标签**：确保标签在小屏幕上清晰可见
- **验证**：错误信息在小屏幕上正确显示

### 3.3 卡片
- **布局**：在小屏幕上使用单列布局
- **内容**：确保卡片内容在小屏幕上不会溢出
- **图片**：使用 `img-responsive` 类确保图片响应式

### 3.4 模态框
- **尺寸**：在小屏幕上使用全屏模态框
- **内容**：确保模态框内容在小屏幕上可滚动

## 4. 字体适配

### 4.1 字体大小
- **移动端**：使用较小的基础字体大小
- **桌面端**：使用较大的基础字体大小
- **响应式调整**：使用CSS变量和媒体查询

### 4.2 行高
- **移动端**：使用更紧凑的行高
- **桌面端**：使用更宽松的行高

## 5. 图片适配

### 5.1 响应式图片
```html
<img src="image.jpg" alt="描述" class="img-responsive">
```

### 5.2 图片加载
- **懒加载**：使用 `LazyImage` 组件
- **占位符**：使用骨架屏或占位图
- **分辨率**：提供不同分辨率的图片

## 6. 性能优化

### 6.1 移动端优化
- **减少资源**：针对移动端提供更小的资源
- **延迟加载**：非关键资源延迟加载
- **缓存策略**：使用适当的缓存策略

### 6.2 触摸优化

## 11. 常见问题

### 11.1 布局问题
- **内容溢出**：使用 `overflow: hidden` 或 `word-break`
- **对齐问题**：使用Flexbox或Grid实现对齐
- **间距问题**：使用统一的间距系统

### 11.2 性能问题
- **加载缓慢**：优化资源加载
- **交互卡顿**：减少JavaScript执行时间
- **内存泄漏**：及时清理事件监听器

### 11.3 兼容性问题
- **浏览器差异**：使用特性检测和polyfill
- **设备差异**：测试不同设备
- **操作系统差异**：考虑不同操作系统的特性
